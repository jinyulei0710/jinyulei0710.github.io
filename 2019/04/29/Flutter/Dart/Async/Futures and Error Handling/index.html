(译) Futures 和错误处理

发表于 2019-04-29 | 更新于 2019-06-25 | 分类于 Flutter 开发 | 阅读次数：

原文链接

本指南介绍了在编写异步代码时如何处理错误。你可以通过两种方式编写异步代码：

  • 使用 async 函数和 await 表达式，其背后使用的是 Futures。
  • 直接使用 Future API。

注意：如果你不熟悉 Futures 背后的一般概念，请先阅读异步编程：Futures。

为什么要使用 Future API 而不是使用 async 和await ？当你需要比从 async 和 await 获得的还要有更多控制时。例如，假设你希望在 future 完成时发生某些事情，但你也希望立即继续你正在做的事情。如果使用 await ，则活动将暂停，直到 Future 完成。使用 Future API ，你可以注册回调并继续。
